# bear-support · 找支持文献

一个任务：你交来一个观点，它带回真实存在的支持文献。支持来自检索，不来自记忆。

先读 `references/sci-cli.md`——CLI 检测步骤、用法和铁律。
再读 `references/output-system.md`——三层输出体系（TUI / Markdown / HTML）和 HTML 外壳规范。

## 说明

`references.bib` 默认生成，包含本次报告所有引用文献。段落模式自动触发，无需加 flag。

## 步骤（固定顺序）

**Step 0 — 确认 CLI 可用**
运行 `sci --version`。未安装就给安装命令并停止；认证错误就提示 `sci init` 并停止。不要跳过。

**Step 1 — 提取主张，识别段落模式**
从一句话取核心断言。如果输入是含多个独立主张的段落，自动进入段落模式，把每个主张拆开逐一处理。

自动进入段落模式时，告诉用户："检测到 N 个独立主张，分别检索。"然后直接开始，不等待确认。

给每个主张稳定编号：`C1`、`C2`、`C3`。后续查询、证据表和未支持主张都要回指这些编号，避免段落模式下混淆。

**Step 2 — 检索，并展示查询词**
对每个主张执行 `sci search`，使用 `--mode low`，`--limit 10–20`，`--prefix` 与主张挂钩。**无论主张是否模糊，都在输出里展示实际使用的查询词**——用户看到"它搜了什么"才能判断结果是否可信，也方便日后自己调整。

如果主张含义确实模糊（如专有名词有多个完全不同的含义），先用一句话确认再执行。

**Step 3 — 读结果、筛支持候选**
读 `.json`，筛出真正支持该主张的候选文献：标题、作者、年份，以及一行"这篇文献如何具体支撑这个观点"（不是摘要复述，是支撑关系）。

筛选时使用四级口径：
1. 直接支撑：研究对象、变量、方向和结论都贴近主张
2. 部分支撑：只支撑机制、人群、条件或某个子主张
3. 间接相关：只能作为背景，不足以支撑核心断言
4. 排除：主题相近但不能支撑，不进入证据表

只把前三类写入证据账本；排除项只在必要时写入检索透明度。

**Step 4 — 按支撑强度排序**
在每个主张内部：直接证据排最前，相关或部分支持排后。对后者诚实标注"间接支持"或"部分相关"。标注最强的那篇——它是用户最应该优先引用的。

如果没有直接支撑，但有部分支撑，首屏结论必须说清楚"只有部分支撑"，不要把部分支撑包装成强支撑。

**Step 5 — 没有就说没有**
如果检索结果里没有真正支持该主张的文献，直接说"未找到支持文献"并标记该主张。不要把弱相关的文献硬拉进来充数。

## 输出格式

每篇一块，直接支撑排最前，间接支持排后：

```
Smith et al. 2023 — "论文标题"
支撑强度 ████████░░ 强
→ 这篇如何具体支持你的主张（一句话）
查询词：sleep memory consolidation
```

强度对照：强 = `████████░░`，中 = `█████░░░░░`，弱/间接 = `███░░░░░░░`
间接支持在强度后加注 `（间接）`。

未找到支持的主张：`[!] [主张] — 未检索到支持文献`，单独一行，不混入列表。

末尾一行：**首选引用：** Smith et al. 2023 — 理由一句话

## 输出物

检索结束后，在当前目录下创建 `{topic-slug}/` 子文件夹，输出四个东西：

1. **终端** — 直接 print 已定义的 TUI 格式
2. **`report.md`** — 按 `output-system.md` 的结构化 Markdown 规范生成：稳定 YAML front matter + 一眼结论 + 证据阶梯 + 研究动作建议 + 证据表 + 详细证据 + 检索透明度 + 可复用 JSON
3. **`report.html`** — 自包含 HTML，按 `output-system.md` 规范生成
4. **`references.bib`** — 合并所有 query 生成的 `.bib` 文件，去重后只保留报告中实际引用的文献条目

HTML 签名体验：**证据阶梯**，强调色 `#16a34a`（绿）。文献区按「直接支撑 / 部分支撑 / 间接相关」分 tabs 展示。HTML 组件规范见 `output-system.md` 的 bear-support 节。

简单查询只给 1–2 条轻量下一步。科研写作场景给具体建议：首选引用放哪、哪些主张需要降调、下一轮补查什么。

## Gotchas

- **部分支撑不要包装成直接支撑**：只支撑机制、子人群或背景的文献，在首选引用和首屏判断中都要如实标注局限，不要省略。
- **段落模式下主张编号要贯穿全文**：C1/C2/C3 编号一旦分配，后续证据表、未支持主张、下一步建议都必须回指这些编号，不要在输出中途改变指代方式。
- **空结果不能删除**：某个主张搜不到支持文献是一个信号，必须在报告里保留，不要因为"看起来更完整"而省略。
