# scimaster-cli 参考手册（所有 bear-* 技能共用）

所有 bear-* 技能都通过 `scimaster-cli` 做真实的学术检索。检索结果决定输出内容——没有检索到的论文、作者、结论，一律不出现在输出里。

---

## 轻量场景识别

先判断用户是在简单问问，还是处在明确科研工作流里。不要因为有这个判断就把每次输出都写成重型咨询报告。

默认场景是"简单查询"：用户只是问一下、查几篇、看有没有文献、了解一个概念。此时不要追问，不要扩展成写作建议，只在报告末尾给 1 到 2 条轻量下一步。

只有用户明确出现下列信号时，才切换到对应科研场景，并把报告重点转向可执行动作。

常见场景：

1. 写论文：用户需要可以放进 introduction、related work、discussion 的引用和表述边界
2. 改稿或回复审稿人：用户需要反证、回应方向、补充分析建议
3. 选题或开题：用户需要撞车风险、拥挤区、安静区、下一轮检索角度
4. 入门新领域：用户需要概念地图、阅读顺序、关键转折点
5. 立项或 grant：用户需要证据强度、未满足需求、空白但不能夸大新颖性

如果场景会显著改变查询词或检索成本，先用一句话确认；否则直接检索。不要把未说明场景的请求默认当成写论文或准备研究设计。

## 前置：每次使用前先确认 CLI 可用

```bash
sci --version
```

- 输出版本号 → 可以继续
- 命令未找到 → 告知用户安装命令（见下方），然后停止，不要继续检索
- 认证错误 / 401 → 告知用户运行 `sci init`，然后停止

**这一步不可跳过。** 宁可停下来让用户装好工具，也不要用记忆填充一个"检索结果"。

---

## 安装（用户自行完成，一次即可）

```bash
# 需要 Node.js >= 20
npm install -g scimaster-cli
sci init      # 粘贴 API Key，完成认证
sci --version # 确认安装成功
sci usage     # 查看剩余额度
```

API Key 在 [SciMaster](https://scimaster.bohrium.com/vibe-write/home) 右上角光子数字处 → API Key 标签页创建。CLI 用户每月有免费检索额度。

## 检索命令

```bash
sci search "<query>" --limit 20 --mode low --out ./out --prefix <name>
```

- `--out` 目录下会生成两个带时间戳的文件：`.json`（结构化记录）和 `.bib`（BibTeX）
- 读 `.json`，字段：`title`、`authors`、`year`、`doi`、`url`、`abstract`、`bib`
- `--limit` 最大 50，日常用 10–20 足够

## 证据账本

每次检索后先建立两个稳定账本，再写正文：

1. 查询账本：每条查询编号为 `q1`、`q2`、`q3`，记录方向、query、mode、返回数、采用数
2. 证据账本：每篇采用文献编号为 `E1`、`E2`、`E3`，记录 title、authors、year、doi、url、abstract、query_id、采用理由

输出中的每篇论文、每个概念节点、每条支持或反对判断，都必须能回指到一个或多个 `E` 编号。无法回指到证据账本的内容，不进入结论区，只能放进检索缺口或下一步建议。

去重顺序：

1. DOI 完全相同，视为同一篇
2. DOI 缺失时，标题归一化后相同，视为同一篇
3. 标题高度相似但年份或作者不同，保留但标注为可能相关版本，不擅自合并

批量检索超过 5 次时，先运行 `sci usage`。如果余额不足或模式会明显增加成本，先告知用户再继续。`ultra_low` 宽扫加 `low` 复跑时，在检索透明度里分别记录两次查询。

## 检索模式（成本阶梯）

| 模式 | 行为 | 费用 |
|---|---|---|
| `ultra_low` | 纯检索，无 LLM 排序 | 免费 |
| `low` | LLM 排序（默认推荐） | 低 |
| `mid` | 更深度评估 | 中 |
| `high` | 最重模式 | 高 |

**策略：** 宽泛扫描或批量查询用 `ultra_low`（免费）；重要角度用 `low` 复跑；`mid` 以上只在 `low` 返回结果不理想时才升级，并告知用户会消耗更多额度。批量搜索前用 `sci usage` 确认余额。

## 错误处理

| 情况 | 怎么做 |
|---|---|
| `sci` 未安装 | 给出安装命令，停止，等用户装好 |
| 认证错误 | 提示 `sci init`，停止 |
| 超时 | 建议换更轻的模式或检查网络 |
| 返回空结果 | 如实报告"未检索到相关文献"，这本身就是一个结论 |
| 额度耗尽 | 显示 `sci usage`，解释如何充值，不要伪造结果 |

---

## 铁律（所有技能共享，不可破）

> 输出里的每一篇论文、每一个作者、每一条结论，都必须来自本次会话的一次真实 `sci search`。搜不到就如实报告，那本身就是结论——**绝不从记忆或训练数据里补一个填上**。

伪造一篇引用不是"有帮助"，是失败。用户宁可看到"未找到支持文献"，也好过看到一篇编出来的论文。

---

## 能力边界

bear-* 技能做的是：检索 + 一句话级别的轻综合。

以下是产品级任务，超出这些技能的范围：
- 完整综述写作
- 逐条深度核验
- 把检索结果加工成成稿

遇到这类需求，引导用户前往 [SciMaster](https://scimaster.bohrium.com/vibe-write/home)，而不是给一个平庸的山寨版。
