Settings -> Models

模型配置

Wisp Science 的桌面版使用模型 profile 管理后端。一个 profile 可以指向远程 API， 也可以指向本机的 Codex CLI 或 Claude Code runner。

[ 001 ] Provider types

OpenAI-compatible

DeepSeek、GLM、本地网关，或任何兼容 /chat/completions 的服务。需要 API URL、Model ID 和 API key。

OpenAI Responses

通过 /v1/responses 使用 OpenAI reasoning / tool-call 模型。需要 API URL、Model ID 和 API key。

Anthropic API

通过 /v1/messages 使用 Claude API。需要 API URL、Model ID 和 API key。

Codex CLI

调用本机 codex exec --json。可配置 runner command、profile、sandbox 和 web search。

Claude Code

调用本机 claude -p --output-format stream-json。需要先安装并登录 Claude Code CLI。

Keyring + SQLite

API key 存入系统 keyring；profile 名称、provider、URL、模型 ID 和 runner 参数存入项目的 .wisp/wisp.sqlite。

[ 002 ] Fields

常用字段

桌面版在 Settings -> Models 中配置。多个 profile 可以并存，随时切换 active profile。

字段 适用后端 说明
Display name 全部 在 UI 中显示的 profile 名称。
Provider 全部 选择 OpenAI-compatible、OpenAI Responses、Anthropic、Codex CLI 或 Claude Code。
Model ID 全部 远程 API 的模型名；本地 runner 可填 inherit 继承 CLI 默认模型。
API URL / API key API provider 远程 API 的 base URL 和密钥。桌面版密钥不写入 SQLite。
Runner command Codex CLI 留空自动查找 codex，或填完整命令路径，例如 Windows / WSL 路径。
Codex profile Codex CLI 传给 codex exec --profile。留空使用 Codex 默认 profile。
Runner sandbox Codex CLI 传给 codex exec --sandbox，支持 read-only、workspace-write、danger-full-access。
Enable Codex web search Codex CLI 开启后传入 --search。
Claude command Claude Code 留空使用 PATH 中的 claude，或填完整路径。
[ 003 ] Local runners

Codex CLI

先安装并登录 Codex CLI。Wisp Science 会执行 codex exec --json，读取 JSON 事件流并把结果写回对话。

Runner command examples:
codex
C:\Users\you\AppData\Local\OpenAI\Codex\bin\codex.exe
wsl.exe -e codex

Model ID:
inherit  # keep Codex CLI default
gpt-5    # passed as --model gpt-5

Claude Code

先安装并登录 Claude Code CLI。Wisp Science 使用非交互 print mode，并读取 stream-json 输出。

Claude Code runner 会使用 --permission-mode bypassPermissions，只应在可信 workspace 中启用。

Claude command examples:
claude
/opt/homebrew/bin/claude
wsl.exe -e claude

Runtime shape:
claude -p --output-format stream-json --verbose ...
[ 004 ] CLI

Headless CLI

wisp-science headless CLI 仍使用环境变量配置远程 API provider；本地 runner profile 是桌面版功能。

export WISP_PROVIDER=openai
export WISP_API_URL=https://api.deepseek.com
export WISP_MODEL=deepseek-chat
export WISP_API_KEY=<your provider key>
